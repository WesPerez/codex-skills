# 证据清单

当视频流程可能要在用户机器上执行时，使用这份清单。

## 来源捕获

- 视频 URL、规范 URL、标题、作者、发布日期、时长。
- 简介文本和所有链接。
- 置顶作者评论、作者回复、最新相关评论、高赞相关评论。
- 评论日期，以及评论是否晚于视频发布。
- 字幕/转写稿、自动字幕、硬字幕、相关弹幕/直播聊天。
- 资源链接、密码、镜像、版本说明、失效或删除的链接。

## 视觉捕获

- 开始时的整页截图或播放器截图。
- 粗间隔总览拼图。
- 每个 UI 转换、文件/路径/弹窗附近的补充帧。
- 裁剪并放大的字幕区、命令区或路径区。
- 下载前的资源列表截图。
- 修改前的本机路径截图或文件列表。
- 如果 UI 状态重要，执行每一步后的截图。

## 流程还原

每一步都记录：

- 时间戳。
- 来源：视频帧、字幕、作者评论、关联 readme、本机验证。
- 动作。
- 目标路径、应用、账号、网站或设备。
- 所需文件名和来源。
- 预期可见状态或结果。
- 风险：只读、下载、本地状态变化、破坏性、安装/运行二进制、凭据/账号、外部副作用。
- 是否需要确认。
- 执行者：Codex、用户，或根据用户选择二者择一。
- 工具：命令行、脚本、结构化 API、Browser、Computer Use 或人工鼠标键盘。
- 是否占用桌面焦点，以及是否依赖前一步的人工作业结果。
- 验证方式和停止条件。
- 未知点或路径/版本不匹配。

## 执行分流

- 记录用户选择：混合高效模式或全托管 UI 模式。
- 先列出并完成不依赖人工结果的 Codex 代理项。
- 把人工明显更快、插件低可靠或只能由用户完成的动作合并成最少批次。
- 浏览器操作交给官方 Browser/Chrome 插件；只有桌面应用才交给 Computer Use。
- Computer Use 可能抢焦点前必须告知用户；用户正在使用电脑时不得突然接管。
- 全托管模式遇到登录、验证码、支付、UAC/安全桌面或连续失败时，明确交还人工，不盲目重试。

## 人工批次清单

每个人工批次都检查是否写明：

- 开始前的应用、窗口、设备、目标路径和前置状态。
- 按钮或菜单的准确中英文名称、图标或相对位置。
- 单击、双击、右键、输入、选择、拖拽等具体动作。
- 完整输入值、文件名和路径。
- 每一步的等待条件和预期可见结果。
- 不应点击、删除、覆盖或授权的对象。
- 报错、界面不一致或结果异常时的停止条件。
- 用户完成后应返回的步骤编号、结果、报错原文或截图。

## 执行安全

- 未经用户确认具体文件和目的，不运行第三方二进制文件。
- 未经明确确认，不卸载、不删除、不替换可执行文件、不编辑注册表、不改安全/隐私设置、不改应用数据。
- 官方下载能满足目标时优先官方来源。
- 可行时先备份原文件再替换。
- 不要因为目录名相似就推断它是同一个目标。
- 如果真实机器和视频明显不同，停止并重新判断。

## 最终审计

报告：

- 哪些内容只是读取。
- 下载了什么、路径在哪里、是否保留。
- 修改、创建、删除或备份了什么。
- 打开或保留了哪些标签页、窗口、进程。
- 清理了什么，以及为什么能证明清理安全。
- 哪些内容因为归属或安全性不清楚而故意没有碰。
- 使用了哪种执行模式，Codex、用户、Browser 和 Computer Use 分别完成了什么。
- 是否发生模式切换，以及为什么切换或交还人工。
