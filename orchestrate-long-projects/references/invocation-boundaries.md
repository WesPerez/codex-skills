# 调用边界与特殊场景

本文件只在无法判断“普通上下文续接”还是“持久项目恢复”时读取。默认不调用长期编排技能。

## 核心区别

```text
普通上下文续接
= 读取旧会话、摘要、handoff、Git 或文件后继续当前任务

持久项目恢复
= 需要独立于会话保存和核验项目停点、共享写入、证据或未知外部动作
```

任务持续时间、是否换了会话、是否发生意外中断，都不能单独证明需要长期编排。

## 决策表

| 场景 | 使用本技能 | 处理方式 |
|---|---:|---|
| 上个普通会话意外中断，新会话读取旧消息后继续 | 否 | 读取旧会话或摘要，按普通任务继续 |
| 上个任务因 token、网络、客户端关闭或用户暂停而结束 | 否 | 先恢复上下文；没有持久项目风险就不建台账 |
| 上次留下未提交代码，下一会话继续修改 | 否 | 查看精确 `git status/diff` 和相关文件，正常实现 |
| 上次测试、构建或 lint 中断，需要重跑 | 否 | 判断命令副作用后直接重跑或定位失败 |
| 上次启动了本地开发服务 | 通常否 | 核对 PID、命令、端口和归属；只有它属于活动长期项目状态时才进入 |
| 只阅读、总结、评审或继续完善旧计划 | 否 | 直接读取计划；用户明确要建立长期台账时再显式调用 |
| 同一任务跨天，但始终在一个会话且状态清楚 | 否 | 使用当前会话/plan；时间长不是触发条件 |
| 新会话拿到一份简单 handoff，继续单一代码改动 | 否 | 读取 handoff 和代码即可 |
| 仓库有活动长期台账，但当前请求是无关小修复 | 否 | 不读写长期台账，不改变其 STATUS |
| 仓库有活动长期台账，当前请求明确推进其当前切片 | 是 | 由项目规则或用户显式调用，fast-first 恢复 |
| 用户明确要求从现在开始建立跨会话可恢复项目 | 是 | 显式调用后选择 light/standard/hardened |
| 多 Agent 只做独立只读研究 | 否 | 普通子代理汇总，不因并行本身建台账 |
| 多 Agent 跨阶段写共享项目并需交接 | 是 | 使用单写入器和明确 handoff |
| 上次可能已发消息、写数据库、发布或调用非幂等 API，但结果未知 | 是 | 只读对账，标记 unknown，禁止重放 |
| 只是下载、浏览或只读数据库查询中断 | 通常否 | 直接重新确认目标和结果；存在共享状态或重复风险时再显式升级 |
| 用户只问活动长期项目的真实进度 | 可选 | 只有用户显式调用或项目规则要求时使用只读 status；否则直接读取 STATUS 回答 |

## 模糊表达

用户只说下面这些话时，不自动调用：

- “继续上次。”
- “上一个会话断了。”
- “新开个会话读之前的内容。”
- “接着之前的规划做。”
- “昨天没做完。”
- “帮我恢复一下。”

先读取用户指定的旧会话、摘要、handoff 或文件，判断是否存在活动台账、共享写入交接或未知非幂等动作。没有就按普通任务继续。

## 已有台账的仓库

活动台账只约束属于该长期项目的切片，不接管仓库内所有后续请求。

进入前必须把当前请求映射到 `STATUS.md` 的目标、phase、slice 或明确 blocker。无法映射时视为无关任务，不运行 resume-check、不更新事件、不创建 checkpoint。

如果项目规则写得过宽，例如“本仓库所有任务都使用长期技能”，应收窄为“只有明确属于 STATUS 当前项目的请求使用”。

## 意外中断但没有台账

没有台账不等于需要补建台账。先使用普通只读恢复：

1. 读取旧会话/摘要和用户指定文件。
2. 查看精确 Git 变化、测试输出和本任务可能启动的进程。
3. 判断是否存在未知非幂等外部动作。

若只是代码、文档、测试或本地服务状态，继续普通任务。只有用户显式要求以后持续维护，或发现未知外部动作/跨阶段共享写入时，才调用本技能；恢复审计通过前不追写虚构历史。

## 不可降级的安全例外

普通续接不使用本技能，但仍必须遵守通用安全规则：数据库身份不明按生产处理、生产禁止代写、未知外部动作禁止盲目重放、清理和停止操作必须证明归属。这些规则不依赖本技能是否启用。
