# LINUX DO 研究审计模板

读取 LINUX DO 内容时在内部使用此模板。它防止过度声称，并让最终报告更精确。

## 范围

```text
用户请求：
目标标签页/标题：
目标 URL：
请求范围：
  [ ] 网络搜索/reader 优先
  [ ] 收藏页
  [ ] 指定主题
  [ ] 所有可见楼层
  [ ] 所有回复/楼层
  [ ] 链接教程
  [ ] 附件/下载
  [ ] 派生交付物
超出范围或延后：
```

## 网络阶段

```text
使用的查询：
是否使用代理：
尝试过的直连 URL：
尝试过的 Reader URL：
尝试过的主题位置 URL：
尝试过的网络 JSON：
网络覆盖：
需要浏览器兜底的网络缺口：
```

不要把搜索摘要当证据。记录原始主题文本，或把结论标为未解决。

## 浏览器标签页

```text
初始打开标签页：
id | 标题 | URL | 归属

接管的现有标签页：
id | 标题 | URL | 原因

创建的临时标签页：
id | 用途 | 请求 URL | 最终 URL | 最终标题 | 已关闭 | 证据

最终打开标签页：
id | 标题 | URL | 保留原因
```

归属值：

- `user-owned`
- `task-created`
- `ambiguous`

只关闭 URL/标题证据匹配的 `task-created` 标签页。

不要把浏览器/插件标签页导航到 LINUX DO 主题 `.json` URL。如果需要浏览器兜底，使用普通主题页和主题位置 URL。

## 收藏覆盖

```text
收藏来源 URL：
提取方法：
可见收藏总数：

收藏索引 | 标题 | URL | 分类/标签 | 摘要 | 已选择？ | 原因 | 状态
```

状态值：

- `read`
- `skipped-out-of-scope`
- `deferred`
- `failed`

除非用户授权过滤，否则不要因为旧收藏看起来旧就跳过。

## 主题覆盖

```text
主题 URL：
标题：
预期帖子/最高楼层：
尝试过的提取方法：
  [ ] 主题 JSON
  [ ] 当前 DOM
  [ ] 临时 /floor 标签页
  [ ] 其他

已读楼层：
缺失楼层：
缺口由什么补齐：
剩余缺口：
```

楼层表：

```text
楼层 | 作者 | 提取来源 | 文本摘要 | 链接 | 附件 | 警告 | 可操作性
```

可操作性值：

- `instruction`
- `warning`
- `attachment`
- `tutorial`
- `question`
- `noise/thanks`

## 链接覆盖

```text
来源楼层 | 链接文本 | URL | 动作 | 结果 | 临时标签页 ID | 标签页已关闭
```

动作值：

- `opened-read`
- `downloaded`
- `resolved-cdn`
- `skipped-profile`
- `skipped-navigation`
- `deferred`
- `failed`

```text
来源楼层 | 可见名称 | 可见大小 | 论坛 URL | 解析 URL | 本地路径 | 字节数 | 已检查 | 笔记
```

检查摘要：

```text
归档路径：
条目数：
JSON 条目数：
格式：
顶层键：
凭据键：
缺失必需字段：
是否打印秘密值：no
```

## 最终报告清单

答复前：

- [ ] 已报告网络方法和代理/直连路径；
- [ ] 没有临时标签页仍保持打开；
- [ ] 未关闭用户拥有的标签页；
- [ ] 下载内容已用绝对路径列出；
- [ ] 生成文件已用绝对路径列出；
- [ ] 覆盖情况已精确说明；
- [ ] 未读链接/楼层/收藏已披露；
- [ ] 未提取 cookies/localStorage/sessionStorage；
- [ ] 未执行论坛状态修改；
- [ ] 清理行为已带证据报告；
- [ ] 如果使用了子代理，已整合其审计摘要。
